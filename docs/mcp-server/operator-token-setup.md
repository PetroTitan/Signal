# Operator token setup

External operators authenticate against `/api/mcp` with a workspace-scoped bearer token. Tokens are minted from `/settings/mcp/tokens` and shown to the operator **exactly once**.

## Format

`sigt_<43 base64url chars>` — total 48 characters. Generated by `webcrypto.getRandomValues(32 bytes)` + base64url encoding.

Signal stores:

- `token_hash` — SHA-256 of the plaintext (unique index for O(1) lookup).
- `token_preview` — the first 8 chars of the random portion (for UI fingerprinting only; never sufficient to authenticate).

The plaintext is **never** stored, never logged, and only returned in the create response.

## Scopes

| Scope | What it unlocks |
| --- | --- |
| `workspace:read` | Read workspace, settings, demo flag |
| `products:read` | List products |
| `products:write_pending` | Create products as pending_review |
| `accounts:read` | List growth accounts |
| `accounts:write_pending` | Create accounts as pending_review |
| `weekly_plans:read` | Read current plan + items |
| `weekly_plans:write_pending` | Create plan items as draft |
| `contracts:read` | Read active weekly contract |
| `execution:read` | Read execution queues + logs |
| `execution:dry_run` | Read dry-run logs for a queue/item |
| `verification:run` | Read latest verification runs |
| `imports:prepare` | Record an import-mapping request |
| `reports:write` | Submit an operator-side report |

Five scopes are explicitly blocked and cannot be selected:

```
publishing:live · social_accounts:create · secrets:read
database:unrestricted · billing:write
```

## Setting up

1. Open Signal as a workspace owner or admin.
2. Visit `/settings/mcp/tokens`.
3. Enter a name (e.g. `claude-code-local-laptop`), pick scopes, optionally set an expiry.
4. Click **Create token**.
5. Copy the plaintext token from the green panel. It is shown only once.
6. Configure your assistant (see [./claude-code-config.md](./claude-code-config.md) / [./codex-config.md](./codex-config.md)).

## Rotating

- Revoke the old token from `/settings/mcp/tokens` (the row stays for audit; status flips to `revoked`).
- Create a new token with the same scopes.
- Update the assistant config.

## Server env

The MCP HTTP bridge needs `SUPABASE_SERVICE_ROLE_KEY` set on the server (never bundled to the client). When unset, `/api/mcp` returns 503 honestly and the `/settings/mcp` server status card says **not configured**.
